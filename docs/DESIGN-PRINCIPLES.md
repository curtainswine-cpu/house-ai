# Design principles

This app is built for an ADHD brain and an Autistic partner. These rules guide every
decision — if a change breaks one of these, it's probably the wrong change.

## For ADHD
- **Externalise memory.** If it's not on screen, it doesn't exist. Nothing important
  hidden behind menus.
- **Today-first.** The home screen shows only what's relevant *right now*, not a giant
  backlog that creates overwhelm.
- **Tiny friction to capture.** Add a chore or expense in ~2 taps, before the thought
  is gone.
- **Immediate reward.** Checking something off is satisfying and visible (the tick
  fills, the count goes up).
- **No punishment.** Missed things wait quietly. Nothing turns red or nags. Shame kills
  motivation.

## For Autism
- **Predictable layout.** Things stay where they are. The same actions are always in
  the same place.
- **Explicit, not implicit.** Routines can be broken into clear written steps. Labels
  say exactly what they mean.
- **Low sensory load.** Muted, warm colours. No flashing, no sound, gentle motion only.
- **Reduce surprises.** You can see what's coming (time-of-day grouping, weekly day).
- **Respects system settings.** Honours "reduce motion" and dark mode automatically.

## Shared / practical
- **Mobile-first**, but works on any screen up to a wall tablet.
- **Big touch targets** (minimum 44–48px) so it's easy to hit.
- **Both of us are first-class.** Every item can be "Kirsten", "Jack", "Either" or
  "Both", with consistent colour coding.
- **Accessible:** real buttons, labels, focus states, high-enough contrast.

## A quick test before adding anything
1. Does it add overwhelm or remove it?
2. Could a tired, distracted brain still use it in one go?
3. Is it predictable — does anything jump or move unexpectedly?
4. Does it judge the user? (It shouldn't.)
